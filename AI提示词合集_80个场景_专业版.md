# AI办公提示词合集 · 80个场景

> 适用：ChatGPT / DeepSeek / 通义千问 / Claude 等主流大模型
> 直接复制 `【】` 中的内容替换为你的实际信息即可使用
> 每类提示词配实战技巧，帮助提升输出质量

---

## 使用快速指南

### 提示词黄金公式

好提示词 = 角色设定 + 上下文 + 具体任务 + 输出格式

```
例：你是【政务信息化专家】，结合【当前政策背景】，为【项目名称】写一份【技术方案概述】，格式为【三段式：背景-方案-预期效果】
```

### 让输出更好的技巧

| 问题 | 追加指令 |
|------|----------|
| 太啰嗦 | "缩短到300字以内，只留结论" |
| 太正式 | "语气改成口语化，像跟同事说话" |
| 缺细节 | "请展开说明第2点，给具体案例" |
| 不具体 | "请用数据和建议代替评价" |
| 格式不对 | "请用表格/列表/对比方式呈现" |

## 一、写材料类（25个）

### 基础写作

1. **写周报**
   "请根据以下要点，帮我写一份本周工作周报，格式为：本周完成、存在问题、下周计划。要求：突出量化成果，避免空话。要点：【粘贴你的要点】"

2. **写简报**
   "请将以下长文提炼成一份300字以内的简报，适合给领导阅读。结构要求：核心结论→关键数据→工作建议。原文：【粘贴原文】"

3. **写会议纪要**
   "以下是会议录音转文字，请帮我整理成标准会议纪要，格式为：会议时间、参会人、会议议题、讨论内容、决议事项（逐条）、待办任务（含责任人和截止时间）。会议内容：【粘贴内容】"

4. **润色公文**
   "请帮我润色以下这段公文，使其更加规范、正式、符合政府行文风格。要求：①保留原意 ②修正语病 ③优化逻辑层次 ④添加适当公文过渡词。原文：【粘贴文字】"

5. **写汇报PPT大纲**
   "请为我写一份关于【汇报主题】的汇报PPT大纲，受众为【领导层/评审专家】。包含：封面、背景/现状、存在痛点、方案介绍、预期成效、实施路径、总结。每页配一句话核心观点。"

6. **写致辞稿**
   "请帮我写一份领导在【活动名称】上的致辞稿。时长3分钟（约600字）。要求：①高站位但不空洞 ②有具体数据和案例 ③结尾有号召力。活动背景：【简述活动】"

7. **写请示文件**
   "请帮我写一份关于【项目/采购名称】的请示文件。要素：请示事由（政策依据+实际需要）、采购内容、预算金额及来源、建议采购方式。语气正式、理由充分。"

8. **写工作方案**
   "请帮我写一份【工作名称】的工作方案。包含：工作目标（量化）、工作任务（分点）、责任分工（部门+人）、时间安排（甘特图方式）、保障措施。要求：可执行、可考核。"

9. **内容压缩**
   "请将以下内容压缩到200字以内，只保留：核心结论、关键数据、建议。删除：背景铺垫、评价性语言、重复内容。原文：【粘贴原文】"

10. **拟写通知**
    "请帮我写一份关于【会议/活动名称】的正式通知。要素：会议时间、地点、参会人员、签到要求、议程安排、注意事项。语气规范，使用公文格式。"

### 进阶写作

11. **写调研报告**
    "请为我设计一份关于【调研主题】的调研报告完整框架，包含：背景与目的、调研方法、调研对象、现状分析、发现的问题、改进建议、附件清单。每节标注应包含的核心要素。"

12. **写项目验收总结**
    "请根据以下信息写项目验收总结报告：项目名称【】、承建单位【】、建设周期【】、建设内容【】、完成情况【】、交付成果【】、系统运行情况【】、用户评价【】。突出数据对比和预期达成率。"

13. **写新闻稿**
    "请根据以下事实写一篇新闻稿：事件：【】、时间：【】、主要人物/单位：【】、核心成果：【】、意义：【】。要求：①标题吸引人 ②导语一句话概括 ③正文按重要度递减 ④适合官网/公众号发布。"

14. **政策文件提炼**
    "请将以下政策文件提炼为5条核心要点，要求：每条不超过50字，按重要性排序，附政策文件核心数据（如有）。文件：【粘贴政策摘要】"

15. **拟写函件**
    "请代写一份关于【事项】的函，发往【收文单位】。语气正式、得体，按公文格式排版。内容说明：【简要描述事由和诉求】"

16. **写专项报告**
    "请根据以下要点，写一份专项工作报告，主题：【】。结构要求：工作背景→主要做法→取得成效→存在问题→下一步计划。每部分配具体数据和案例。"

17. **写整改方案**
    "针对以下问题【列出问题清单】，请撰写一份整改方案。包含：问题表现→原因分析→整改措施→责任部门→完成时限。措施要具体可操作。"

18. **写评估报告**
    "请帮我写一份关于【项目/系统名称】的评估报告框架。包含：评估背景、评估范围、评估指标（建议每项配权重）、评估方法、评估结论、改进建议。"

19. **写可行性分析**
    "请为【项目名称】写一份可行性分析纲要。从以下维度分析：政策可行性、技术可行性、经济可行性、管理可行性、风险分析。每个维度给出评估结论。"

20. **拟写合同条款说明**
    "请为我解释以下合同条款的含义和注意事项，用通俗语言。合同条款：【粘贴条款原文】"

21. **写操作手册**
    "请为【系统/工具名称】写一份用户操作手册，面向【初级/高级】用户。结构：功能概述→操作步骤（配截图注释）→常见问题→联系方式。步骤要分步编号。"

22. **写申报材料**
    "请帮我整理【申报项目/课题】的申报材料框架。包含：申报背景、研究内容、技术路线、预期成果、创新点、经费预算。突出创新性和可行性。"

23. **写制度文件**
    "请草拟【制度名称】（如：《数据安全管理制度》）。结构：总则→组织职责→具体管理规定→监督考核→附则。参照政府文件的格式和用语。"

24. **写会议发言稿**
    "请帮我写一段【会议名称】上的发言稿，角色为【角色】，时长5分钟。要求：开门见山，重点突出，有数据支撑，结尾有呼吁或承诺。"

25. **写经验材料**
    "请根据以下工作成效，整理一份经验交流材料。主题：【工作名称】。结构：背景→主要做法（要有特色亮点）→取得成效→经验启示。语言要有感染力。"

## 二、标书方案类（15个）

26. **写技术方案全景**
    "你是政府信息化解决方案架构师。请为【项目名称】写一份完整的技术方案概要，需包含：建设目标、总体架构（含分层图描述）、核心功能模块、关键技术路线、信创适配方案。风格专业、详实。"

27. **写项目背景（政策版）**
    "请为【项目名称】写项目背景。要求：①从国家层面政策切入【如：数字中国、数字政府、AI规划】②落地到省/市政策【如：浙江省数字经济、宁波市数字化改革】③结合实际业务痛点。每点引一个具体政策文号。"

28. **写技术路线说明**
    "请描述本方案采用的技术路线，包括：后端框架【】、前端框架【】、数据库【】、AI平台【】、容器方案【】。每个选项需说明选型理由，突出：先进性、成熟度、信创适配性、生态丰富度。"

29. **评分标准逐条响应**
    "以下是招标文件中的评分标准：【粘贴标准原文】。请帮我撰写逐条响应内容。要求：①每条都使用对应的小标题 ②直接回应评分点 ③用'针对……我方提出……'句式 ④每个响应点不少于200字。"

30. **写团队介绍**
    "请写一份项目团队介绍模板。角色：项目经理、技术负责人、AI专家、实施工程师。每人格式：姓名→角色→资历（年限+类似项目经验）→相关证书。格式专业，适合标书。"

31. **写实施计划**
    "为【项目名称】写一份详细实施计划。分阶段描述：一、需求调研（2周）→二、系统设计（2周）→三、开发实施（8周）→四、测试部署（2周）→五、试运行（4周）→六、验收（2周）。每阶段写明工作内容、交付物、关键里程碑。"

32. **写培训方案**
    "请为【系统名称】设计用户培训方案。培训对象分三类：系统管理员、业务操作员、领导层。每类说明：培训目标→培训内容→培训方式→学时→考核方式。"

33. **写售后服务承诺**
    "请写一份正式的售后服务承诺书。包含：①服务响应分级标准（P1-P4级别及时效）②服务方式（电话/远程/现场）③服务团队配置 ④质量保障措施 ⑤培训服务。语气正式可信，凸显责任感。"

34. **写安全方案章节**
    "请写一份系统安全设计方案，符合国家等保三级要求。涵盖：物理安全、网络安全、主机安全、应用安全、数据安全五个层面，每层列出3-5条具体措施。"

35. **写创新性说明**
    "请写一段项目创新性说明，从以下角度展开：①技术创新【如AI、大数据】②模式创新【如流程再造】③管理创新【如决策支持】。要求：每条创新点都有具体应用场景描述。"

36. **写需求理解章节**
    "针对招标文件中的需求描述【粘贴需求】，写一份'需求理解'章节。结构：①用户现状分析 ②核心痛点 ③我们的理解 ④应对思路。体现对业务的深入理解。"

37. **写数据治理方案**
    "请写一份数据资源建设与治理方案。包含：数据资源梳理、数据标准制定、数据质量管控、数据共享交换、数据安全保障。每个部分列出具体措施。"

38. **写持续性保障**
    "请写一份系统运维保障方案。包含：日常巡检（周期+内容）、故障处理（流程+时效）、版本更新（频率+方式）、应急预案（场景+措施）、定期报告（频次+内容）。"

39. **写预算编制说明**
    "请为【项目名称】写预算编制说明模板。费用分类：硬件设备、软件采购、开发实施费用、第三方服务费、培训费、运维费。每类给出编制依据和估算方法。"

40. **写验收标准和测试方案**
    "请为【项目名称】设计验收标准和测试方案。包含：①功能验收清单（按模块逐项）②性能指标（并发数/响应时间/吞吐量）③安全测试（漏扫/渗透/等保）④验收文档清单。"

## 三、数据分析类（8个）

41. **数据发现**
    "请分析以下数据，指出：①3个关键发现（要具体）②2个异常点（如有）③2条可落地的建议。数据：【粘贴数据或描述】"

42. **图表选型**
    "我有以下数据需要可视化：【描述数据结构和目的】。请建议最佳图表类型并说明理由。选项：折线图/柱状图/饼图/散点图/热力图/雷达图/漏斗图。"

43. **多维对比**
    "请将A和B进行多维度对比分析，用表格呈现。对比维度包括但不仅限于：功能、性能、成本、易用性、生态。A：【描述】; B：【描述】。"

44. **写分析报告**
    "请根据以下数据写一份分析报告。结构：概述（一句话核心结论）→关键发现（3-5条，每项配数据）→趋势分析（同比/环比）→问题诊断→建议措施。数据：【粘贴】"

45. **趋势预判**
    "根据以下过去12个月的数据，分析趋势模式，预测下3个月的可能范围，并标注置信度。数据：【粘贴数据（时间序列）】"

46. **异常检测**
    "以下数据集可能存在异常值，请帮我：①标出可能的异常点 ②给出判断依据 ③建议处理方式。数据：【粘贴数据】"

47. **KPI看板设计**
    "请帮我设计一张管理者看板的指标框架。关注领域：【列出关注方向】。建议指标不超过8个，每个指标标注：名称、计算公式、数据来源、更新频率、预警阈值。"

48. **数据报告自动化**
    "请帮我写一段Python代码，自动从【数据源】读取数据，计算【指标】，生成【图表类型】，保存为【输出格式】。"

## 四、沟通类（10个）

49. **写正式邮件**
    "请帮我写一封发给【收件人/单位】的正式邮件。主题：【主题】。内容要点：【列出要点】。语气【正式/温和/有力】。"

50. **回复投诉**
    "请帮我写一封回复投诉的邮件。既要诚恳认错，又要给出明确解决方案。投诉内容：【描述投诉】。我方改进措施：【列出措施】。"

51. **写催办邮件**
    "请帮我写一封催办邮件，提醒【对方】尽快完成【事项】。要求：礼貌但有力度的表达紧迫性，给出具体截止时间。"

52. **写推荐信**
    "请帮我写一封推荐信，推荐【姓名】担任【职位/角色】。重点突出3个核心优势，配具体事例。"

53. **写感谢信**
    "请帮我写一封感谢信，感谢【对方/单位】在【具体事件】中的支持和帮助。语气真诚，避免套话。"

54. **写合作邀请函**
    "请帮我起草一份合作邀请函。邀请【对方单位】参与【项目/合作名称】。说明合作背景、合作方式、双方预期收益、下一步安排。"

55. **写会议邀约**
    "请帮我写一则会议邀约，邀请【参会人】参加【会议名称】。包括：会议主题、时间地点、预计时长、议程亮点。让收信人有参加的意愿。"

56. **写项目汇报邮件**
    "请帮我写一封项目进展汇报邮件。面向【领导/客户/团队】。包括：当前进展（完成率%）、里程碑达成情况、风险与问题、下阶段计划。简洁清晰。"

57. **写跨部门协调函**
    "请帮我写一份跨部门协调的正式函件。事由：【需要协调的事项】。要求：说明需要对方配合的具体内容、原因、时间节点、我方可提供的支持。"

58. **写工作总结**
    "请根据以下信息，写一份季度/年度工作总结。结构：核心成绩（用数据说话）→主要工作（分点）→存在不足（诚实但有改进思路）→下季度重点计划。信息：【提供信息】。"

## 六、创意辅助类（10个）

71. **项目命名**
