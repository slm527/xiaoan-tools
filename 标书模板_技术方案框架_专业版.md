# 使用说明

## 模板使用三步法

1. **替换**：将文中所有 `【】` 标记替换为实际项目信息
2. **裁剪**：根据评分标准删除不需要的章节
3. **丰富**：在每个章节填入项目具体内容

## 评分黄金法则

| 评分项 | 建议分值占比 | 聚焦要点 |
|--------|:----------:|----------|
| 对需求的理解 | 15-20% | 必须逐条响应招标需求，多用"针对…我们提出…"句式 |
| 技术方案完整性 | 30-40% | 6+1层架构（业务/应用/数据/技术/安全/部署+集成） |
| 先进性/创新性 | 10-15% | AI+大数据+信创三个关键词必不可少 |
| 实施保障 | 10-15% | 甘特图+WBS+风险矩阵，体现项目管理能力 |
| 项目团队 | 5-10% | 人员资质证书扫描件+PMP/软考/高级职称 |
| 售后服务 | 5-10% | 分级响应时效（7×24小时 / 4小时到场 / 1小时响应） |

## 高分写作技巧

- **逐条对应法**：每一条招标需求都要有明确的方案章节回应
- **数字说话法**：能用数字绝不用形容词（"系统并发支持500用户">"系统并发能力强"）
- **风险预判法**：主动提出项目可能的风险及应对措施，体现专业性
- **政策引用法**：每章开头引用1-2个国家/省市政策文件作为依据
